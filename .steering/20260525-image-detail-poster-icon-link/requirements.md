# 要求内容: 画像詳細ページの投稿者アバターをリンク化する

## 背景

- 関連 Issue: [#147 画像詳細ページの投稿者欄の画像もリンクにする](https://github.com/kakikubo/lgtmhub/issues/147)
- 画像詳細ページ (`/images/{id}`) の投稿者欄は、現在「投稿者名 (テキスト)」だけが GitHub プロフィールへのリンクになっており、隣接するアバター画像はリンク化されていない。
- ユーザーは投稿者名・アイコンのどちらをクリックしても投稿者のプロフィールに飛べると期待する。

## 要求

1. 画像詳細ページの投稿者欄のアバター画像 (アイコン) もクリックすると、投稿者名と同じく投稿者の GitHub プロフィールに遷移すること。
2. 既存の名前リンク (新規タブで `https://github.com/{login}`) と同じ遷移先・同じ挙動 (`target="_blank"`, `rel="noopener noreferrer"`) であること。
3. プロフィール取得失敗時 (`data-fallback="true"`, Unknown 表示) は、現在と同じくリンクを張らないこと。
4. スクリーンリーダー / キーボード操作のアクセシビリティが現状から悪化しないこと。
   - 既存仕様: アバターは装飾扱い (`alt=""`)、テキスト側に意味のあるリンクが張られている。
   - 同じ宛先のリンクが 2 つ並ぶとスクリーンリーダーで重複読み上げ・Tab 移動の冗長化が起きるため、アバター側のリンクは支援技術 / フォーカスから除外する。

## 対象外

- 画像一覧ページ (`/`) の `ImageCard` は本タスクの対象外 (一覧側に投稿者情報はそもそも出ない)。
- 投稿者プロフィールの実装変更や `resolveUploaderDisplay` のデータ整形ロジックには手を入れない。
- `UploaderProfileRow` のレイアウト・スタイルは原則維持し、リンク化に必要な変更のみ加える。

## 完了条件

- `components/uploader-profile-row.tsx` でアバター画像をリンク化している。
- フォールバック時はリンクを張らないことが視認可能 (`data-fallback="true"` ケースのアバターは依然 `a` タグでラップされない)。
- 既存の E2E (`tests/e2e/image-detail.test.ts`) を更新し、アバター側にも GitHub プロフィールへのリンクが張られていることを検証している。
- `npm test` / `npm run lint` / `npm run typecheck` がすべてグリーン。
