# 要求: 画像にコピーボタンを用意する (Issue #169)

## 背景

現状、一覧画面 (トップページ) では各画像カードの**下部**に「マークダウンをコピー」ボタンが常時表示されている (`components/image-card.tsx` → `components/copy-markdown-button.tsx`)。

[lgtmoon](https://lgtmoon.herokuapp.com/) のように、画像へマウスオーバーしたときにコピーボタンが現れる UX にしたい。あわせて画像下部の常時表示ボタンを削除し、一覧の見た目をすっきりさせる。

## 受け入れ条件

1. 一覧画面の各画像カードで、画像下部に常時表示されていたコピーボタンを廃止する。
2. 画像にマウスオーバー (hover) すると、画像の上にコピーボタンがオーバーレイ表示される。
3. オーバーレイのコピーボタンを押すと、これまで同様マークダウン (`![LGTM](url)`) がクリップボードへコピーされ、コピー完了フィードバックが表示される。
4. コピーボタンのクリックで画像詳細ページへ遷移しない (リンクと競合しない)。
5. キーボード操作 (Tab フォーカス) でもコピーボタンに到達・操作できる。

## スコープ

- **対象**: 一覧画面 (`components/image-card.tsx`)。
- **対象外**: 画像詳細ページ (`app/(site)/images/[id]/page.tsx`) のコピーボタンは現状維持。
- **対象外**: コピー内容の仕様変更 (マークダウン形式のまま。「リンクのみコピー」への変更は別関心事)。

## 非機能・制約

- 既存パターン (data-testid, Tailwind, `cn` ユーティリティ) を踏襲する。
- 影響範囲を最小化する (共有コンポーネント `CopyMarkdownButton` は後方互換を保つ)。

## 既知の制約

- hover を持たないタッチデバイスではオーバーレイは表示されない。モバイルでは画像タップで詳細ページへ遷移でき、詳細ページ下部のコピーボタン (常時表示) からコピーできるため、コピー手段は確保される。
