### 学んだこと

1. **定数はリテラルで再現せず import する**: `HOME_IMAGES_CACHE_TAG` の値を推測で書いて 1 度失敗した。実装と同じ定数を参照すれば、値の変更にテストが自動追随し、そもそも推測が発生しない
2. **カバレッジの低さは「測れていない」ではなく「書かれていない」ことがある**: 34.88% の正体は POST ハンドラのテストが 1 本も無いことだった。計測範囲を直した（#255）ことで初めて可視化された欠落であり、可視化の価値が具体的に現れた事例
3. **残る未カバー分岐は到達不能なフォールバック**: Branch 85.71% の残りは `?? '入力値が不正です'` 等。zod が常に message を返すため到達させられない。ここを埋めるために実装を歪めるべきではない
