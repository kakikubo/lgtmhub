# requirements.md — 一覧画面にランダム表示機能を追加する (Issue #109)

## 背景

一覧画面 (`app/(site)/page.tsx` → `HomeContent`) は `created_at` 降順 (新着順) で
16 枚 (#108 で導入した `LIST_IMAGES_DEFAULT_LIMIT`) を表示し、カーソルベースの
「もっと読み込む」で追加取得する。新着順だと過去の画像が埋もれ、ユーザーが
多様な画像に出会いにくい。

依存 Issue #108 は main にマージ済み (`LIST_IMAGES_DEFAULT_LIMIT = 16`)。

## 目的

ページ先頭にランダム表示ボタンを配置し、押下するとサーバーが全 active 画像から
ランダムに 16 枚を抽出して表示する。

## 要求仕様

### ボタン
- 配置: ページ先頭 (一覧グリッドより上、見出し付近)。
- 文言: 「ランダム表示」。
- 常時表示。総件数が 16 件以下でも押下可能 (全画像をランダム順表示)。

### ランダムの範囲
- サーバーサイドで全 active 画像からランダムに 16 枚を抽出
  (表示中 16 枚のクライアントシャッフルではない)。
- 16 は #108 の共通定数 `LIST_IMAGES_DEFAULT_LIMIT` を参照 (ハードコード禁止)。

### ランダム表示中の挙動
- 16 枚で完結。「もっと読み込む」は表示しない (カーソルは `created_at` 基準で
  ランダム順と不整合)。
- ボタン再押下で別の 16 枚を再抽出 (再シャッフル)。

### 状態の永続性
- ランダム表示は一時的なクライアント状態。リロード・再訪問で自動的に
  通常表示 (新着順 16 枚 + もっと読み込む) に戻る。
- 「通常表示に戻す」ボタンや URL への状態保持は設けない (最小構成)。

### 既存挙動への影響
- 初回ページ表示は従来どおり新着順 16 枚 (#108)。ランダムはボタン押下時のみ発動。
- empty / error state は既存ハンドリングを流用する。

## 受け入れ条件

- [ ] ページ先頭に「ランダム表示」ボタンが表示される。
- [ ] 押下すると全 active 画像からランダムな 16 枚が表示される。
- [ ] 再押下で別の 16 枚が表示される (前回と異なる組み合わせになり得る)。
- [ ] ランダム表示中に「もっと読み込む」が出ない。
- [ ] リロードすると新着順 16 枚 + もっと読み込む (#108 の挙動) に戻る。
- [ ] 抽出件数が #108 の共通定数を参照している。
- [ ] E2E (`tests/e2e/image-list.test.ts`) にランダム表示の検証を追加。

## 非対象 (Out of Scope)

- 「通常表示に戻す」専用 UI / URL クエリへの状態保持。
- ランダム表示でのページネーション。
- ランダム結果での投稿者プロフィール (アバター) 表示
  (既存 `LoadMoreButton` の追加グリッドも profiles 無しで描画する前例に倣う)。
