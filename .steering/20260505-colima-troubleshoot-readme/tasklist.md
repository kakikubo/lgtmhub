### 学んだこと

**技術的な学び**:
- supabase の `[analytics]` セクションは `enabled = false` にすると `supabase_vector` が起動対象から外れるため、colima でのソケットマウント失敗を回避できる。デフォルトを変えずに README 側で「ローカル一時無効化」を案内するのが最も影響範囲が小さい。
- `npm run lint` は RTK 経由だと出力が「ESLint output (JSON parse failed: ...)」と表示されるが、実体は `biome lint .` 。原因切り分けには `./node_modules/.bin/biome lint .` を直接叩くのが確実。

**プロセス上の改善点**:
- ドキュメント変更だけのタスクでも、`implementation-validator` のレビューで強調表現の改善提案(blockquote 切り出し)が拾えたので、文書 PR でも同サブエージェントを通すのは有効だった。
- ステアリングディレクトリ作成と並行して、`requirements.md` で「採用案 A を選択した理由」と「スコープ外（案 B/C）」を最初に明文化したことで、実装中の迷いが発生しなかった。
