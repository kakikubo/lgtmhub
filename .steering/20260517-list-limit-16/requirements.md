# 要求: 一覧画面の1ページ表示件数を16枚に制限する

関連 Issue: https://github.com/kakikubo/lgtmhub/issues/108

## 目的

一覧画面の1ページあたり表示件数を **20件 → 16件** に変更する。
16件を超える分は既存の「もっと読み込む」ページネーションの対象とする。

## 仕様

### ページネーション方式
- 既存のカーソルベース「もっと読み込む」方式を維持する。番号付きページ送り・無限スクロールへの変更は行わない。
- 変更点は1ページの取得件数のみ（20 → 16）。

### 件数定数
- `LIST_IMAGES_DEFAULT_LIMIT` を 16 に変更し、16 を単一の共通定数（single source of truth）として定義する。
- 初期表示（`getHomeImagesInitial`）と「もっと読み込む」（`/api/images` のデフォルト limit）の両方がこの定数を参照すること。
- この共通定数はランダム表示機能（別Issue）からも参照される前提とする。

### 既存挙動の維持
- 並び順は `created_at` 降順（新着順）のまま。
- 総件数が16件以下のときは `nextCursor` が `null` となり「もっと読み込む」ボタンは表示されない。
- empty / error state は既存のまま。

## 受け入れ条件

- [ ] 一覧画面の初回表示が16件になっている。
- [ ] 17件以上ある場合、「もっと読み込む」で次の16件が取得できる。
- [ ] 16件以下のとき「もっと読み込む」ボタンが出ない。
- [ ] 16 が共通定数として一箇所で定義され、初期表示と API デフォルトの双方が参照している。
- [ ] 既存 E2E（`tests/e2e/image-list.test.ts`）が引き続きパスする。
