## このドキュメントの位置づけ

このドキュメントは壁打ち・ブレストの成果物で、正式な仕様書ではありません。

## プロダクトビジョン

githubなどでコードレビューをする際、LGTMコメントするだけでは味気ない。
気の利いた画像付きでLGTMである事がわかれば、作業者も喜ぶ筈。

このサイトから画像コピーボタンを押せば、LGTM画像のリンクをマークダウンで得る事ができ、Approve時に好きに貼る事ができる。
ユーザがログインする事ができ、ログインするとユーザは独自の画像を生成する事ができる。その画像にLGTMの文字を入れる事ができる。


**一言で言うと**: lgtmoon.com の ~~パクリ~~ オマージュ

## 解決したい課題

### 今困っていること

1. **lgtmoon.comのLGTM文字が白文字一択なので画像によってはみえづらい**
   - lgtmoon.comはとてもいいサイトなのだが、**LGTMの文字が白文字一択**なので画像によってはLGTMと書いてあるのかどうかが**分かり辛い**
   - せっかく画像を貼っているのに何をいわれたんだかわからない感じになってしまう

2. **不適切なコンテンツ**
   - あきらかにアダルト画像がはってあったりする
   - 会社で開こうもんならその手の部署に通報される
   - **ユーザがログインしなくてもLGTM画像を生成できてしまう**のがそもそもの間違いなのではないか

3. **ホゲータが多すぎる**
   - ポケットモンスターにでてくる「ホゲータ」とその亜種の画像を大量生成した人がいるらしく、かなりの確率で「ホゲータ」に遭遇する
   - 飽きた
   - そもそも「ホゲータ」知らないし、好きじゃない
   - **同じ画像を何度も生成できてしまう**ロジックがもう全然いけてないと思う。
   - 自動化できそうなのに、GUIのツールだとうまく連携できない

## 主要機能の候補

### P0 (MVPで絶対必要)

#### 1. 画像の登録機能

ユーザはweb上に存在する画像のURLを登録できる。
LGTMhub上では「LGTM」という文字（白文字・黒縁固定）を画像の上に合成し、新たなLGTM画像としてCDNに保存する。

- 登録時にサーバーサイドで画像をダウンロード・合成・CDN保存（Sharp使用）
- 対応フォーマット: JPEG・PNG・GIF（静止画のみ）、最大10MB
- 保存時にWebP形式に変換・幅1200px以内にリサイズ
- 重複チェック: 画像ハッシュで既存画像と照合し、一致する場合は作成せず既存画像ページへリダイレクト
- 登録制限: 1ユーザーあたり1日10枚まで
- 登録処理は非同期（処理中表示→完了通知）
- ファイルアップロードはP1以降（MVPはURL登録のみ）

#### 2. 画像の削除機能

ユーザは自身が登録したLGTM画像を削除できる。

#### 3. 認証機能

GitHub OAuthを行い、ログインさせることができる。
GitHubのアバター・表示名を取得してプロフィールに使用する。

閲覧・Markdownリンクのコピーはログインなしで可能。
画像の登録のみログイン必須。

#### 4. お気に入り登録

ユーザは登録されている画像をお気に入り登録できる。
自身が登録した画像でなくとも問題ない。
お気に入りリストは自分のみ閲覧可能（非公開）。

#### 4.1 お気に入り画像の一覧画面

ユーザはお気に入り登録した画像だけを一覧表示する画面を持つ。

#### 5. 画像一覧画面

- デフォルト表示順: 新着順（登録日時の降順）
- ログインなしでも閲覧・Markdownリンクのコピーが可能

### P1 (重要だけど後回しでもいい)

#### 6. 管理者による画像削除機能

ユーザが登録した画像の中に問題があるものが含まれていると判断された場合、管理者の権限で削除することができる。

#### 7. ユーザー通報機能

ユーザは不適切と思われる画像を通報できる。
通報された画像は管理者に通知され、確認・削除の対象となる。

#### 8. ファイルアップロード対応

URL登録に加え、ローカルファイルからの画像アップロードに対応する。

### P2 (将来的に検討)

- ランダム表示機能
- アニメーションGIF対応

## 差別化ポイント

### 既存ツールとの比較

| 機能 | lgtmoon.com | LGTMhub |
|------|-------------|---------|
| LGTM文字スタイル | 白文字固定（見づらいケースあり） | 白文字+黒縁固定（視認性確保） |
| 画像生成 | ログインなしで可（大量生成可能） | ログイン必須（乱用防止） |
| 重複画像 | 制限なし | 画像ハッシュで重複排除 |
| 不適切コンテンツ | 対策なし | 管理者削除＋ユーザー通報機能 |
| 登録数制限 | なし | 1日10枚/ユーザー |
| お気に入り | なし | あり（非公開） |

### このツールの強み

- **安心安全**: ログイン必須の登録＋通報機能により、会社で開いても問題ないコンテンツ品質を維持
- **視認性**: 白文字+黒縁のLGTM文字で、どんな画像でも文字が読める
- **重複排除**: 同じ画像の大量登録を防ぎ、一覧の多様性を保つ

## 技術的な検討事項

### データの保存方法

- **フレームワーク**: Next.js
- **DB・認証**: Supabase（PostgreSQL）
- **CDN・ホスティング**: Vercel（Vercel Blob）
- **画像処理**: Sharp（サーバーサイドでLGTM文字合成・WebP変換）

### GitHub APIとの連携

- GitHub OAuthによるログイン認証
- GitHubアバター・表示名の取得（プロフィール表示用）
- それ以外のAPI利用はなし

## 非機能要件

### パフォーマンス

- 画像一覧の表示: 3秒以内
- 画像登録処理（ダウンロード・合成・CDN保存）: 10秒以内（非同期処理）
- Markdownリンクのコピー: 即時

### 使いやすさ
- 初めて使う人でも5分で基本操作を覚えられる
- ログインなしでも閲覧・コピーが可能（導入障壁を低く）
- エラーメッセージが分かりやすい

### 信頼性
- データが消えない（自動バックアップ）
- エラーが起きたら元に戻せる
- 危険な操作には確認を求める

### モバイル対応
- PCファースト設計
- レスポンシブ対応はするが、モバイル最適化は後回し

### 収益化
- 完全無料で運営（Vercel・Supabaseの無料枠を活用）
- 収益化はMVP成立後に検討
